# 02 — Embeddings + built-in RAG

Embed a tiny local corpus into a named RAG workspace and run semantic search —
fully offline after the first model download. No vector DB to operate yourself.

**Tested against `@qvac/sdk` 0.15.0 · Node ≥ 20**

## What you'll learn
- Load an embeddings model (`GTE_LARGE_FP16`) with `loadModel` and `onProgress`.
- Ingest documents into a named workspace with `ragIngest`, then search with `ragSearch`.
- Skip re-embedding when the workspace already exists (`ragListWorkspaces`) — resource reuse.
- Free the model with `unloadModel` in a `finally` block; close the workspace without deleting it.

## Requirements

| Requirement | Value |
|-------------|-------|
| Runtime | Node.js 20+ |
| Model | `GTE_LARGE_FP16` |
| Download | approximately 0.67 GB |
| Free RAM | approximately 1–1.5 GB |
| Input | five bundled UTF-8 text files |

## Run it

From the repository root:

```bash
cd recipes/02-embeddings-rag
npm install
npm start
```

Ask your own question by passing it through:

```bash
npm start -- "What makes sourdough bread rise?"
```

First run downloads the model and embeds the sample corpus (progress shown).
Subsequent runs see workspace `cookbook-02-embeddings-rag` via `ragListWorkspaces`
and skip ingest (data stays on disk after close unless you delete it).

## Expected output

Document IDs and similarity scores are generated by the local workspace and may
differ between SDK versions. The default espresso query should rank the
espresso document first:

```text
Ingesting 5 docs into workspace "cookbook-02-embeddings-rag"…

Query: How is espresso coffee made under pressure?

1. score=…  id=…
   Espresso …

2. score=…  id=…
   …
```

## How it works
`src/index.ts` is commented as the tutorial. In short: `loadModel` downloads/loads
GTE Large and returns a `modelId`. The recipe lists on-disk workspaces; if
`cookbook-02-embeddings-rag` is missing it calls `ragIngest` with the five short
`sample-data/` files (`chunk: false` because they are already tiny). One
`ragSearch` prints the top-3 hits with `score` and a content snippet — the default
query is about espresso, so that doc should rank first. `ragCloseWorkspace` releases
locks but keeps the store; `unloadModel` in `finally` frees model memory.

## Troubleshooting
- **No logs?** Set `QVAC_CONFIG_PATH` to a JSON file containing
  `{ "loggerLevel": "info", "loggerConsoleOutput": true }` and re-run.
- **Out of memory / load failure?** This embeddings model wants ~1–1.5 GB free RAM.
- **Want a fresh ingest?** Close with delete, e.g. in a one-off:
  `await ragCloseWorkspace({ workspace: "cookbook-02-embeddings-rag", deleteOnClose: true })`
  (or `ragDeleteWorkspace` after the workspace is closed).
- **First run hangs at 0%?** The initial download needs network; check connectivity.

## Going further
- Pass a different query on the CLI and watch which sample doc ranks first.
- Try `chunk: true` with longer documents and inspect how hits fragment.
- Need an external vector DB? Use the segregated path: `ragChunk` → `embed` →
  `ragSaveEmbeddings` (or store vectors yourself). Built-in RAG is the simpler default.
